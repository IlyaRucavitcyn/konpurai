Reporting Dashboard

The Reporting Dashboard serves as a comprehensive solution for generating and managing detailed reports for your AI governance activities. This system creates reports using information from your Compliance, Assessment, and Vendor/Risk sections, providing you with actionable insights and documentation for regulatory compliance and internal auditing purposes.

Overview

The Reporting Dashboard consists of two main components:

  • Report Generation: Create new reports with customizable parameters and framework-specific options
  • Report Management: View, download, and manage previously generated reports

Report Generation

To generate a new report, follow these steps:

  1. Click "Generate your report": This button opens the report generation modal
  2. Select Project: Choose the project for which you want to generate the report (required)
  3. Choose Framework: Select the compliance framework (EU AI Act or ISO standards)
  4. Select Report Type: Choose from available report types based on your selected framework
  5. Name Your Report: Provide a descriptive name for your report
  6. Click "Generate report": The system will process your request and automatically download the report

Available Report Types

The system offers different report types depending on the selected framework:

EU AI Act Framework Reports:

  • Project risks report: Comprehensive analysis of project-specific risks and their mitigation strategies
  • Compliance tracker report: Detailed tracking of compliance status against EU AI Act requirements
  • Assessment tracker report: Progress and results of AI system assessments and evaluations
  • Vendors and risks report: Analysis of vendor relationships and associated risk factors
  • All reports combined in one file: Comprehensive report containing all available report types

ISO Standards Framework Reports:

  • Project risks report: Risk assessment and management documentation following ISO standards
  • Clauses and annexes report: Detailed compliance mapping against specific ISO clauses and annexes
  • Vendors and risks report: Vendor risk assessment and management documentation
  • All reports combined in one file: Complete report package with all ISO-related documentation

Report Generation Process

When you generate a report, the system follows this process:

  1. Validation: The system validates your project selection and framework configuration
  2. Data Collection: Gathers relevant data from Compliance, Assessment, and Vendor/Risk sections
  3. Report Processing: Processes the data according to the selected report type and framework
  4. Automatic Download: The completed report is automatically downloaded to your device
  5. Report Tracking: The generated report is added to your reports list for future reference

Report Management Table

The reports table displays all previously generated reports with the following information:

  • Report Name: The name you assigned to the report
  • Type of Report: The specific report type that was generated
  • Project: The project associated with the report
  • Date Generated: When the report was created
  • Generated By: The user who created the report
  • Action: Available actions for the report (download, delete)

Report Actions

For each report in the table, you can perform the following actions:

  • Download: Download the report file to your local machine
    • Click the actions menu (three dots) for the report
    • Select the download option
    • The report will be downloaded to your default download location
  • Delete: Remove the report from the system
    • Click the actions menu (three dots) for the report
    • Select the delete option
    • Confirm the deletion in the warning dialog
    • Note: This action is non-recoverable

Project Filtering

Use the project filter dropdown to organize and view reports by specific projects:

  • All Projects: View all reports across all projects (default view)
  • Specific Project: Select a project to view only reports associated with that project

The filter dropdown is located above the reports table and allows you to quickly focus on specific project documentation.

Pagination and Navigation

The reports table includes pagination controls for managing large numbers of reports:

  • Reports Per Page: Choose how many reports to display per page (5, 10, 15, 20, or 25)
  • Page Navigation: Navigate between pages using the pagination controls
  • Item Count: View the total number of reports and current page range

The pagination footer shows "Showing X - Y of Z project report(s)" to help you understand your current position in the reports list.

Prerequisites

Before you can generate reports, ensure the following:

  • Projects Exist: You must have at least one project created in the system
  • Framework Configuration: Projects must have appropriate frameworks configured (EU AI Act or ISO standards)
  • Data Availability: Ensure relevant data exists in Compliance, Assessment, and Vendor/Risk sections

If no projects exist, the "Generate your report" button will be disabled, and you'll see the message: "Create a project first to generate a report."

Report Generation Status

During report generation, you'll see different status indicators:

  • Processing: The system is collecting data and generating your report
  • Success: Report generated and downloaded successfully
  • Access Denied: You don't have permission to download the report (403 error)
  • Error: An unexpected error occurred during report generation

Empty State

When no reports have been generated, the system displays a helpful empty state:

  • Visual Indicator: An empty state illustration is displayed
  • Informative Message: "There is currently no data in this table."
  • Guidance: The empty state helps users understand that they need to generate reports to see them in this section

Best Practices

  • Descriptive Report Names: Use clear, descriptive names for your reports to make them easy to identify later
  • Regular Report Generation: Generate reports regularly to maintain up-to-date documentation
  • Project Organization: Use the project filter to organize reports by specific projects or initiatives
  • Framework Selection: Choose the appropriate framework (EU AI Act or ISO) based on your compliance requirements
  • Report Archiving: Consider downloading and archiving important reports for long-term storage
  • Data Maintenance: Ensure your underlying data (compliance, assessments, vendor risks) is current and accurate

Security and Access Control

The reporting system includes several security features:

  • User Permissions: Report generation and access are controlled by user roles and permissions
  • Project-Based Access: Users can only generate reports for projects they have access to
  • Secure Downloads: Reports are generated securely and downloaded directly to your device
  • Audit Trail: All report generation activities are logged for compliance and audit purposes

Troubleshooting

Common issues and solutions:

  • Generate Button Disabled: Ensure you have at least one project created in the system
  • Report Generation Fails: Check that your project has the necessary framework configuration
  • Access Denied Error: Verify you have the appropriate permissions for the selected project
  • Download Issues: Ensure your browser allows downloads and check your download folder
  • Empty Reports: Verify that relevant data exists in your Compliance, Assessment, and Vendor/Risk sections
